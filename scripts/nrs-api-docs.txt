Authentication
Every request needs a Company HTTP header containing your API key. Find it under Admin -> Company Management . POST data goes as JSON in the request body. All responses are JSON.
Basic request
curl -X POST https://www.nrsaccounting.com/api/company \
-H "Company: YOUR_API_KEY" \
-H "Content-Type: application/json"
With POST body
curl -X POST https://www.nrsaccounting.com/api/customer/list \
-H "Company: YOUR_API_KEY" \
-H "Content-Type: application/json" \
-d '{"customerId": 1}'

/apinvoice/save
https://www.nrsaccounting.com/api/apinvoice/save
Creates a new AP invoice batch. Matches vendor by apVendorId or by name. Each detail line requires either glAccount or invStockId.
Required
name: vendor name (looked up by name if apVendorId not given)
externalCode
externalId
invoiceDate
details
Optional
apVendorId: vendor ID (preferred over name lookup)
invoiceNumber: vendor's invoice number
note: invoice description/memo
dueDate
divisionId
jcJobId
taxAmount, taxAccountId
shippingAmount, shippingAccountId
details[].glAccount (or invStockId)
details[].note: line description
details[].quantity (default 1)
details[].unitCost or totalCost
details[].vendorPart: vendor part number
details[].jcJobId
Example Request
{
"externalCode": "MySystem",
"externalId": "INV-001",
"apVendorId": 1,
"invoiceDate": "06/01/2024",
"invoiceNumber": "VI-123",
"note": "June supplies",
"taxAmount": 0,
"shippingAmount": 0,
"details": [
{
"glAccount": "1-500-00",
"note": "Supplies",
"quantity": 2,
"totalCost": 500.00
}
]
}
Example Response
{
"saved": true
}

curl -v https://www.nrsaccounting.com/api/attachment/upload/{lookup} \
-H "Company: YOUR_API_KEY" \
-F "file=@/{filename}"

/expense/save
https://www.nrsaccounting.com/api/expense/save
Submits a mobile employee expense. Creates a checkbook credit-card entry (ckEntryTypeId 50) against the chosen expense account. Attach receipt photos separately via upload with foreignTable=ckCheckReg and the returned ckCheckRegId . Both save and submit are accepted as the action name.
Required
apExpenseAccountId: id from apexpenseaccount/list
amount: must be > 0
Optional
apVendorId: vendor to pay; defaults to NOF (no-vendor) if omitted
name: payee name (used when no vendor or NOF vendor)
address1, address2, address3, city, state, zip
description: defaults to "Mobile Expense - {account name}"
accountId: override the bank/cash account; defaults to company mobile expense setting
Example Request
{
"apExpenseAccountId": 1,
"amount": 45.99,
"description": "Client lunch",
"name": "Corner Cafe",
"address1": "789 Oak Ave",
"city": "Spokane",
"state": "WA",
"zip": "99202"
}
Example Response
{
"saved": true,
"ckCheckRegId": 5001
}
