Authentication

Every request needs a Company HTTP header containing your API key. Find it under Admin → Company Management. POST data goes as JSON in the request body. All responses are JSON.

Basic request

curl -X POST https://www.nrsaccounting.com/api/company \ -H "Company: YOUR_API_KEY" \ -H "Content-Type: application/json"

With POST body

curl -X POST https://www.nrsaccounting.com/api/customer/list \ -H "Company: YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{"customerId": 1}'

/apinvoice/save
https://www.nrsaccounting.com/api/apinvoice/save

Creates a new AP invoice batch. Matches vendor by apVendorId or by name. Each detail line requires either glAccount or invStockId.

Required

  • name: vendor name (looked up by name if apVendorId not given)
  • externalCode
  • externalId
  • invoiceDate
  • details

Optional

  • apVendorId: vendor ID (preferred over name lookup)
  • invoiceNumber: vendor's invoice number
  • note: invoice description/memo
  • dueDate
  • divisionId
  • jcJobId
  • taxAmount, taxAccountId
  • shippingAmount, shippingAccountId
  • details[].glAccount (or invStockId)
  • details[].note: line description
  • details[].quantity (default 1)
  • details[].unitCost or totalCost
  • details[].vendorPart: vendor part number
  • details[].jcJobId

Example Request

{ "externalCode": "MySystem", "externalId": "INV-001", "apVendorId": 1, "invoiceDate": "06/01/2024", "invoiceNumber": "VI-123", "note": "June supplies", "taxAmount": 0, "shippingAmount": 0, "details": [ { "glAccount": "1-500-00", "note": "Supplies", "quantity": 2, "totalCost": 500.00 } ] }

Example Response

{ "saved": true }
